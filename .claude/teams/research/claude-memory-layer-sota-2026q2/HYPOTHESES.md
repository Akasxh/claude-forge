# HYPOTHESES.md — claude-memory-layer-sota-2026q2

**Seeded by**: research-lead (round 0, before investigation)
**Policy**: bias toward boring explanations; 3-4 competing framings; each
must be falsifiable or at least differentiable from the others.

These are seeded BEFORE investigation to prevent confirmation bias and
give the skeptic something to attack.

---

## H1 — Evolving-playbook memory (ACE / retrospector pattern is already SOTA)

**Claim**: The agent memory Akash needs is **not a vector store**. It is
a structured playbook of durable lessons generated by a reflection loop
(generator → reflector → curator), injected into context at session
start. This is the pattern his Research Team already uses: retrospector
writes lessons, scribe curates/dedupes, research-lead reads at session
start. The 2025 ACE paper (arxiv 2510.04618) shows this beats fine-tuning
AND vector-RAG for agent self-improvement on agent benchmarks, finance,
and code tasks.

**Differentiator**: Zero vector embeddings, zero graph database, zero
external services. Pure file-system, pure Claude prose.

**Falsifiable by**: A head-to-head benchmark where an ACE-style playbook
is beaten by a vector-RAG or graph memory on a task that actually
benefits from long-tail fact recall (e.g., 100k-turn agent session with
many unique entities).

**Initial status**: Open — is the strongest prior, because this pattern
is already running and already seeded his MEMORY.md with 7 lessons.

---

## H2 — Temporal knowledge graph (Zep/Graphiti pattern)

**Claim**: The right memory layer is a **temporally-aware knowledge
graph**: entities, relationships, timestamps, and bi-temporal validity
("this fact held from T1 to T2 but was superseded at T3"). Retrieval is
hybrid graph-walk + vector similarity + BM25. Zep's Graphiti open-
sourced this in 2024-2025 and their paper claims +18.5% over MemGPT on
LoCoMo and ~2.58× lower latency.

**Differentiator**: Captures how facts change over time (a critical
failure mode of flat vector memory). Surfaces contradictions.

**Falsifiable by**: Is the benchmark lead real after adversary audits
(Zep's own paper, likely self-interested)? Does it actually matter for
a single-user coding-agent setup where there is no "many users, many
facts about each" dynamic?

**Initial status**: Open — strong in the literature, but the benchmark
provenance is marketing-adjacent (Zep-authored paper about Zep's
product), so adversary pass is mandatory.

---

## H3 — Hippocampal-indexed PageRank memory (HippoRAG 2)

**Claim**: The SOTA for long-term memory is **HippoRAG 2** (arxiv
2502.14802 / OSU NLP, published Feb 2025, accepted ICML 2025). It
combines Personalized PageRank over an open-information-extraction graph
with LLM-powered indexing, modeling the hippocampus's pattern-separation
+ pattern-completion. Reported +7% over standard RAG on associative and
multi-hop QA, matches iterative RAG on factual, and beats it on sense-
making.

**Differentiator**: Beats both flat-vector RAG and graph-only approaches
on all three competence tests of long-term memory (factual, sense-
making, associative). ICML 2025 peer review.

**Falsifiable by**: Does HippoRAG 2 still lead on tasks that look like
"long agent history" rather than "Wikipedia QA"? Its benchmarks are
primarily QA-flavored, not coding-agent-flavored.

**Initial status**: Open — strongest academic pedigree, but task
mismatch is a risk.

---

## H4 — Self-editing in-context memory (Letta / MemGPT pattern)

**Claim**: The right memory layer is **the LLM editing its own context
window via tool calls**. MemGPT/Letta lets the agent explicitly call
`memory_insert` / `memory_replace` / `memory_search` tools, with a
"core memory" block always in context and an "archival memory" searchable
on demand. The key insight is that the LLM learns *what to remember and
when* rather than an external classifier deciding. Letta is an
open-source company incorporated in 2024, ~18k GitHub stars, still
actively maintained as of early 2026.

**Differentiator**: Zero supervision for write decisions; the agent's
own reasoning drives memory updates. No external curator pipeline.

**Falsifiable by**: Does agent-directed memory management actually
outperform a retrospector + scribe pipeline in a real session? Does
it suffer from write-amplification (LLM thinks every fact is important)?
Are Letta's published benchmarks from the original MemGPT paper still
valid on 2026 frontier models?

**Initial status**: Open — architecturally closest to what Akash might
want for per-session Claude coding assistance, but operationally heavier.

## Differential predictions

| Hypothesis | If true, what we'd observe |
|---|---|
| H1 ACE | Short, curated lesson lists beat long retrieved history; retrospector+curator pair is load-bearing; adding a vector store gives marginal improvement |
| H2 Graphiti | Temporal queries surface contradictions that flat vector misses; entity-centric retrieval is a big win for multi-session recall |
| H3 HippoRAG 2 | PPR graph walks outperform both cosine KNN and flat BM25 on multi-hop tasks; LLM-as-indexer is the crucial step |
| H4 Letta | Agent-directed memory operations give better long-horizon coherence than curator pipelines; core/archival split is the load-bearing primitive |

## What we're NOT assuming yet

- That any one of these is "the answer." The real answer may be a
  hybrid (e.g., ACE playbook as hot memory + HippoRAG 2 as cold memory).
- That the best architecture for Claude Code = the best architecture
  in the papers. Claude Code's constraints (subagents cannot sub-
  dispatch, `memory: user` only injects one file, 25KB ceiling) may
  force a hybrid or a reduced subset.
- That benchmark leadership means real-world leadership. All
  benchmark claims go through the adversary.
